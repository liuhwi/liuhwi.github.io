Rsync问题排查

发表于 2010-01-26 | | 阅读次数

错误1

解决：很大可能是服务器端没有开启 rsync 服务。开启服务。

错误2

解决：服务器端同步目录没有权限，cwrsync默认用户是Svcwrsync。为同步目录添加用户Svcwrsync权限。

错误3

解决：配置文件 rsync.conf中添加 lock file = rsyncd.lock 即可解决

错误4

解决：client端没有设置/etc/rsync.pas这个文件，而在使用rsync命令的时候，加了这个参数–password-file=/etc/rsync.pas

错误5

解决：client端已经设置/etc/rsync.pas这个文件，里面也设置了密码111111，和服务器一致，但是服务器段设置有错误，服务器端应该设置/etc/rsync.pas ，里面内容root:111111 ,这里登陆名不可缺少

错误6

解决：服务器端的/data/backup 其中backup这个目录并没有设置，所以提示：chdir failed

错误7

解决：磁盘空间不够，所以无法操作。

错误8

解决：注意查看同步的目录权限是否为755

错误9

解决：连接时间超时，检查服务器的端口netstat –tunlp，远程telnet测试。

错误10

解决：rsync服务未启动，启动rsync服务：rsync –daemon –config=/etc/rsyncd.conf

错误11

解决：Ctrl+C或者大量文件

错误12

解决：查看rsync日志
rsync: unable to open configuration file “/etc/rsyncd.conf”: No such file or directory
xnetid查找的配置文件位置默认是/etc下，根据具体情况创建软链接。例如：
ln -s /etc/rsyncd/rsyncd.conf /etc/rsyncd.conf
或者更改指定默认的配置文件路径，在/etc/xinetd.d/rsync配置文件中.

数据库操作SQL语句

1. 显示Mysql所有数据库

2. 创建名称为test的数据库

3. 删除名称为test的数据库

4. 选择test数据库

表操作SQL语句

1. 显示当前数据库中存在什么表

mysql > SHOW TABLES;

2. 创建数据库表user：在mysql>后粘贴以下SQL语句，存储引擎为MYISAM，字段id为主键、唯一索引。

mysql > CREATE TABLE spitfire (
id INT( 5 ) UNSIGNED NOT NULL AUTO_INCREMENT,
username VARCHAR( 20 ) NOT NULL,
password CHAR( 32 ) NOT NULL,
time DATETIME NOT NULL,
number FLOAT( 10 ) NOT NULL,
content TEXT NOT NULL,
PRIMARY KEY (id)
) ENGINE = INNODB ;

INSERT INTO spitfire (id, username, password, time, number) VALUES (’1′, ‘lhw’, ’123456′, ’2011-1-1′, 8888);

3. 查看user表结构

mysql > DESCRIBE user;

从表中检索信息

1. 从user表中检索所有记录

mysql > SELECT * FROM user;

2. 从user表中检索特定的行

字段username等于abc，字段number等于1，按字段id降序排列

mysql > SELECT * FROM user WHERE username = ‘abc’ AND number=’1′ ORDER BY id DESC;

3. 从user表中检索指定的字段

指定username和password

mysql > SELECT username, password FROM zhangyan;

4. 从user表中检索出唯一的不重复记录

mysql > SELECT DISTINCT username FROM user;

5. 插入信息到user表

mysql >

6. 更新user表中的指定信息

mysql > UPDATE user SET content = ‘hello china’ WHERE username = ‘abc’;

7. 删除user表中的指定信息

mysql > DELETE FROM user WHERE id = 1;

8. 清空user表

mysql > DELETE FROM user;

9. 快速清空表

利用SQL语句清空了该数据表，而不保留日志

mysql > TRUNCATE TABLE TableName

10.删除user表

mysql > DROP TABLE user;

11.更改表结构

将user表username字段的字段类型改为CHAR(25)

mysql > ALTER TABLE user CHANGE username username CHAR(25);

12.将当前目录下的mysql.sql导入数据库

mysql > SOURCE ./mysql.sql;

mysql用户管理

1. mysql修改密码

mysqladmin -uroot -p”oldpwd” password “newpwd”

2. 使用下面语句生成加密密码，之后修改密码为加密后的密码

mysql> SELECT PASSWORD(‘PAWD’);
mysql> SET PASSWORD FOR spitfire@”%” = PAWD；

3. 使用update语句,密码同样为加密后的密码

mysql> UPDATE USER SET password=”newpwd” WHERE user=root and host=localhost;
mysql> FLUSH PRIVILEGES;

权限操作SQL语句

1. 查看用户信息

mysql -uroot -p -e “use mysql;select user,host from user”

2. 创建一个具有root权限，可从任何IP登录的用户leon，密码为123456

mysql > GRANT ALL PRIVILEGES ON . TO ‘leon’@’%’ IDENTIFIED BY ’123456′;

3. 创建一个具有“数据操作”、“结构操作”权限，只能从192.168.1.*登录的用户leon，密码为123456

mysql > GRANT SELECT , INSERT , UPDATE , DELETE , FILE , CREATE , DROP , INDEX , ALTER , CREATE TEMPORARY TABLES , CREATE VIEW , SHOW VIEW , CREATE ROUTINE, ALTER ROUTINE, EXECUTE ON . TO ‘leon’@’192.168.1.%’ IDENTIFIED BY ’123456′;

4. 创建一个只拥有“数据操作”权限，只能从192.168.1.24登录，只能操作test数据库的user表的用户leon，密码为123456

mysql > GRANT SELECT , INSERT , UPDATE , DELETE ON test.user TO ‘leon’@’192.168.1.24′ IDENTIFIED BY ’123456′;

5. 创建一个拥有“数据操作”、“结构操作”权限，可从任何IP登录，只能操作test数据库的用户leon，密码为123456

mysql > GRANT SELECT , INSERT , UPDATE , DELETE , CREATE , DROP , INDEX , ALTER , CREATE TEMPORARY TABLES , CREATE VIEW , SHOW VIEW , CREATE ROUTINE, ALTER ROUTINE, EXECUTE ON test.* TO ‘leon’@’%’ IDENTIFIED BY ’123456′;

6. 创建一个拥有“数据查询”权限，可以从192.168.0.10登陆，只能查询table1和table2表的用户query，密码是zbcx*123456

mysql > GRANT SELECT ON table1. TO ‘query’@’192.168.0.10′ IDENTIFIED BY ‘zbcx123456#’;
mysql > GRANT SELECT ON table1. TO ‘query’@’192.168.0.10′ IDENTIFIED BY ‘zbcx123456#’;

7. 删除用户

mysql > DROP USER ‘spitfire’@’%’;

8. 去除用户权限

mysql > REVOKE ALL PRIVILEGES ON . from ‘query’@’192.168.0.%’;
mysql > REVOKE GRANT OPTION ON . from ‘query’@’192.168.1.%’;

9. 查看用户权限

mysql > show grants for root@’localhost’;

MySQL备份

1. 混合引擎

mysqldump -uroot -p –default-character-set=utf8 –opt –lock-all-tables db_name > db_name.sql

2. 导出全库到一个文件

mysqldump -uroot -p –default-character-set=utf8 –lock-all-tables –flush-logs –all-databases > x.sql
mysqldump -uroot -p –default-character-set=utf8 –hex-blob –routines –lock-all-tables database > /tmp/database.sql
